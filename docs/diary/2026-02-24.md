# 日報 2026-02-24

## 今日やったこと

印鑑登録証明システムのMVP実装を完了した。

### 実装内容

**技術スタック**: React Router v7 + TypeScript + Tailwind CSS (フロントエンドのみ、バックエンドはモック)

#### 作成ファイル

- `app/lib/types.ts` - 型定義（SealRegistration, SealRegistrationStatus等）
- `app/lib/mock.ts` - モックデータとCRUD関数（純粋関数スタイル）
- `app/components/SealPreview.tsx` - 印影SVGコンポーネント（氏名から動的生成）
- `app/components/StatusBadge.tsx` - 登録状態バッジ
- `app/routes/home.tsx` - 検索・一覧ページ
- `app/routes/registrations/new.tsx` - 新規登録フォーム
- `app/routes/registrations/$id.tsx` - 詳細・証明書発行ページ

#### 機能一覧

1. **検索・一覧** (`/`)
   - 登録番号・氏名・住所での絞り込み検索
   - 印影サムネイル付きテーブル表示

2. **新規印鑑登録** (`/registrations/new`)
   - 和暦（元号選択）での生年月日入力
   - 印影リアルタイムプレビュー（SVG、氏名から動的生成）
   - フォームバリデーション
   - 登録前確認ダイアログ

3. **詳細・証明書発行** (`/registrations/:id`)
   - 印鑑登録原票の全項目表示
   - 印鑑登録証明書プレビュー・印刷
   - 廃止（抹消）操作

## 所感

仕様書（国の標準仕様書）が非常に詳細で、実際の行政システムの複雑さを実感した。
MVPとしてコアフローを絞り込んだことで、短時間で動くものが作れた。
印影をSVGで動的生成するアイデアはGeminiとの壁打ちで出てきたもので、UIとしてわかりやすくなった。

## 学び

- React Router v7のファイルベースルーティングでは `$id.tsx` のようなダイナミックセグメントが使える
- モックデータを純粋関数（immutableなstate更新）で管理するとテスタブルになる
- 行政システムは和暦・元号管理が必須で、これをUIでどう扱うかが重要な設計ポイント
